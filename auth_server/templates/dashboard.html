How This SSO System Works (The "EasyAuth" Workflow)

  1. User visits App: The user opens App 1, 2, or 3 and clicks on "Login with SSO"
  2. Redirect to Auth Server: The app redirects the user to this "Auth Server" (this page)
  3. Authentication: The user enters their Email and Password
  4. MFA Verification: If MFA is enabled, the system asks for the 6-digit TOTP code
  5. JWT Cookie: Upon successful login, the Auth Server generates a JWT Token (including the user's 'role') and saves it in a secure cookie
  6. Redirect back to App: The server redirects the user back to the original App
  7. Logged In: App 1 (and 2, 3) reads the shared cookie to validate the session and enforce Role-Based Access Control (RBAC)
  8. Global Logout: Clicking Logout deletes the cookie, instantly logging the user out of all connected apps
